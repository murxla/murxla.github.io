Installation

git clone https://github.com/murxla/murxla.git
cd murxla
mkdir build
cmake ..
make

During the configuration phase (cmake ..) the build system checks whether any of the supported solvers are installed. If a solver was installed to a custom path <path> you can tell the build system by specifying the path as follows:

cmake .. -DCMAKE_PREFIX_PATH=<path>

After successful compilation you can find the Murxla binary in build/bin/. Please refer to the User Guide for how to use Murxla.

Supported Solver

Murxla currently has native integration for the following solvers:

  • Bitwuzla

  • Boolector

  • cvc5

  • Yices

Solvers without native integration can still be tested via the SMT-LIBv2 interface (option --smt2 <solver-binary>).
